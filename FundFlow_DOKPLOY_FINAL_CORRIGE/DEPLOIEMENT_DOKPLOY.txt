FUNDFLOW - REGLAGES DOKPLOY
============================

METHODE RECOMMANDEE : APPLICATION + DOCKERFILE

1. Envoyer LE CONTENU de ce dossier à la racine du dépôt GitHub.
   Sur GitHub, app.py, Dockerfile et nixpacks.toml doivent être visibles directement.

2. Dans Dokploy > Application > Source :
   - Branch : main
   - Build Path : /   (ou laisser vide selon la version de Dokploy)

3. Dans Build Type :
   - Type : Dockerfile
   - Dockerfile Path : Dockerfile
   - Docker Context Path : .
   - Docker Build Stage : vide

4. Dans Environment :
   PORT=8000
   DATA_DIR=/app/data

5. Dans Domains :
   - Container Port : 8000
   - Path : /

6. Supprimer les anciennes variables PORT=5000 ou un ancien Run Command.
   Laisser Build Command et Run Command vides avec le mode Dockerfile.

7. Lancer un déploiement avec cache nettoyé.

METHODE DE SECOURS : NIXPACKS
- Le fichier nixpacks.toml force Python et la commande :
  python -u app.py --no-browser
- Container Port : 8000

PERSISTANCE DE LA BASE
- Monter un volume Dokploy sur : /app/data
- Sans volume, les données SQLite peuvent disparaître lors d'un redéploiement.
